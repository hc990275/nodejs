# 版本更新日志 (Changelog)

所有针对 `kata-new-nodejs` 仓库的功能升级和错误修复记录。

## [4.0.0] - 2026-03-09
### 双平台大一统融合 (Major Feature)

- 🎉 **平台探测自动化引擎 (`index.js` 重构)**: 将本仓库原本单一的 Bash Wrapper 全部推翻。引入智能环境变量嗅探机制（针对翼龙的 `P_SERVER_UUID` 等）。现在 `index.js` 单文件可以在 Lunes Host 容器及普通 Kata VM 下自适应切换安装流，不再需要分离不同的仓库。
- ✨ **环境隔离分发**:
  - 环境识别为 Pterodactyl 面板时（例如 Lunes Host）：采用静默挂载、读取控制台公网分配、绕开一切终端读写直接构建后台死循环进程守护，极大提升在恶劣白嫖云环境下的驻留能力。
  - 环境识别为 Kata 或其他普通 Node 环境时：动态分配五大随机数字端口，并直接接入官方拉取终端链进入交互配置。
- ✨ **Sing-Box 协议演进**: 全部部署废弃 Xray / Websocket 遗留下来的黄字兼容警告（以及被主动探测漏洞），向最前沿的 TCP Reality `xtls-rprx-vision` 流控及 UDP 的最优选 `TUIC` V5。
- 🐛 **修复崩溃顽疾**: 解决了面板常常遇到缺少绑定 IP 和无独立外网给核心传出 `-1` 造成的致命 `SyntaxError` 和配置 `Exit code 1` 循环闪溃挂起，新增强校验跳出算法。

> ℹ️ *备注：历史仅支持单环境旧版本的代码已封存进分支 `kataold` 中。*
